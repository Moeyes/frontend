| Metric | Value |
|---|---|
| Total screens audited | 33 (including 2 confirmed placeholders) |
| Screens passing all checks | ~12 of 31 functional screens |
| Screens with ≥1 critical issue | 19 |

**Top 3 systemic UX issues**

1. **Pagination shows no total count** — every list screen shows `X / Y pages` only; total record count is never visible to users. The i18n key `registration.list.pagination.showing` ("Showing {from} to {to} of {total} participants") is defined but never wired into `DataTable`.
2. **No success toast on admin create/edit** — Users, Organizations, and Sports forms redirect silently after save; the user has no confirmation that the action succeeded.
3. **FLAGGED and PENDING submission badges are visually identical** — both use `variant="outline"`, making them indistinguishable for admins reviewing the queue.

**Khmer rendering health: YELLOW**  
- Battambang font correctly loaded in root layout. All strings have full Khmer translations. Cannot visually confirm tofu-free rendering without browser. One confirmed gap: `CardsPage` uses native `<select>` which renders system font (not Battambang) for its option text on some OS.

**Accessibility health: YELLOW**  
- Good baseline (ARIA labels on icon buttons, `aria-current`, semantic HTML). Critical gaps: pagination `‹`/`›` buttons have no `aria-label`; CardsPage `<select>` elements have no associated `<label>`; DataTable inline empty state is just `—` with no screen-reader announcement; no skip-navigation link in layout.

## 3. Cross-Cutting Issues

These problems appear on **3 or more** screens — they are template/component bugs, not per-screen bugs.

### XC-1 — Pagination counter missing total record count
**Screens affected**: All 7 list screens that use `DataTable` with pagination  
**Detail**: `DataTable` renders `{pageIndex + 1} / {pageCount}`. The `common.showing`/`common.to`/`common.total` i18n keys exist and the registration screen even has a dedicated `registration.list.pagination.showing` key, but `DataTable` never receives or renders a total count. Users cannot see how many records exist, and cannot estimate how many pages to navigate.  
**Root cause**: `DataTable` props interface has no `totalCount` field.

### XC-2 — No success feedback on admin create/edit
**Screens affected**: `/users/new`, `/users/[id]`, `/organizations/new`, `/organizations/[id]`, `/sports/new`, `/sports/[id]` (at minimum)  
**Detail**: Form mutations redirect to the list on success with no `toast.success()`. Users cannot distinguish "saved and redirected" from "navigated away accidentally". Sports delete does show a toast; sports update does not — inconsistent within the same module.  
**Root cause**: `onSuccess` callbacks in mutation hooks call `router.push()` but no `toast.success()`.

### XC-3 — Pagination navigation buttons lack accessible labels
**Screens affected**: All screens with `DataTable` pagination  
**Detail**: The previous/next buttons render bare Unicode characters `‹` and `›` with no `aria-label`. Screen-reader users hear "less-than" or just the character; they cannot identify these as pagination controls.  
**Root cause**: `DataTable` `Button` elements have no `aria-label` prop.

### XC-4 — Search inputs use only `placeholder`, no `<label>`
**Screens affected**: `/users`, `/register`, `/participation` (and any other screen with a bare `<Input placeholder="Search...">`)  
**Detail**: Placeholders disappear on focus and are not announced reliably by screen readers as field purpose. There is no visible or programmatic label.  
**Root cause**: Screens instantiate `<Input placeholder={...}>` without wrapping in a `<label>` or passing `aria-label`.

### XC-5 — Sidebar fixed width, no mobile collapse
**Screens affected**: All portal screens (8+ screens tested at mobile widths)  
**Detail**: `CommonLayout` renders a fixed `w-56` sidebar alongside `flex-1` main content. On 375px viewports the sidebar consumes ~150px of 375px width, leaving ~225px for content. Tables overflow their container. There is no hamburger toggle or responsive collapse.  
**Root cause**: `CommonLayout` has no breakpoint logic for the sidebar.

### XC-6 — Required field markers not shown
**Screens affected**: `/events/new`, `/sports/new`, `/organizations/new`, `/users/new` (all admin create forms)  
**Detail**: All fields use `labelKey` prop to display translated labels. Required fields are not marked with `*` or any visual indicator in the rendered UI. The `common.required` i18n key exists but is not used next to field labels. Users cannot distinguish required from optional without trial-and-error.  
**Root cause**: `TextInputField` has no `required` visual prop; forms do not add `*` to label text.

### XC-7 — No last-updated / data freshness indicator
**Screens affected**: All list and dashboard screens (10+ screens)  
**Detail**: Lists fetched via TanStack Query are cached; there is no indication of when data was last fetched or whether it is stale. Users have no way to know if a list is reflecting the current database state.  
**Root cause**: No `updatedAt` display anywhere in the shared layout or table components.

## 6. Additional Observations (Non-Priority)

### Khmer typography
- Battambang at 400/700 weight is loaded correctly. Line-height is inherited from Tailwind's `leading-normal` (1.5). Khmer script needs ≥1.6–1.8 for descenders. Adding `leading-relaxed` (1.625) to the `body` or a `.font-kh` utility class would improve readability without visual regression on Latin text.
- `CardsPage` uses native `<select>` and `<option>` elements. Khmer text in `<option>` renders in the OS system font on Windows (not Battambang). Replacing with the `SelectField` / shadcn `Select` component would apply the Battambang font via CSS.

### Admin dashboard error handling
`AdminDashboard` shows `{t('failedToLoad')}` as a `text-xs text-muted-foreground text-center` paragraph only when `query.isError`. This renders as tiny grey text easily overlooked. It should use `PageErrorState` to be visually consistent with all other error states.

### Survey admin tab
`SurveyAdminTab` is rendered inside the surveys pages for admin role. It shows submissions by org/sport in a table but has no pagination. For events with many organizations this will be a long scrollable table with no way to filter to a specific org.

### Registration team mode
`/register/team` (CSV bulk upload) is fully implemented and functional. The preview table correctly shows validation status per row. One gap: the "Document notice" column shows `docNote.birthCert`/`idOrPassport`/`canAddLater` but does not render a distinct visual indicator (icon or color) to distinguish "required now" from "can add later" — both are plain text.

### Missing column: Users list org_id
The `users.columns.orgId` i18n key is defined in both `en.json` and `kh.json`, suggesting the intent to show the organization ID in the user list. It is not rendered in `UserList.tsx`. For admin users who need to identify which org a user belongs to, this is a missing data point.

### Submission list: sport column missing
The submissions list displays `sport_event_id` (raw numeric ID) in the detail view but has no sport name column in the list. Admins must click into each submission to see which sport it relates to.

### No breadcrumbs on detail pages
All detail pages use only a `BackLink` to return to the list. On multi-level routes (e.g., `/events/[id]/surveys`) there is no breadcrumb trail showing the current location, making it harder for users to orient themselves.

### Cards page: no pagination
`CardsPage` fetches cards with no pagination. If an event × org combination has hundreds of participants, all cards are rendered at once, which could cause performance issues.

### i18n key duplication
`events.backToEvents` is defined twice in both `en.json` and `kh.json` (lines 145/171 in kh, 145/171 in en). This is valid JSON (last definition wins) but signals a maintenance risk.

### "Sport" column in `participations.columns` defined but unused
The i18n key `participation.columns.sport` is defined but `OrganizerList` renders only 4 columns: name, leaderRole, date_of_birth (mislabeled), actions. Sport is never shown.
