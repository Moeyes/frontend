## 5. Top 10 Priority Fixes

### P1 — Data integrity risk

**FIX-01 · `useCreateOrganizer` cache invalidation namespace**
- **Risk:** After creating an organizer, the organizer list shown in the UI stays stale until next manual refetch.
- **Module:** participation
- **File:** `modules/participation/hooks/useCreateOrganizer.ts:9`
- **Fix prompt:** Replace `qc.invalidateQueries({ queryKey: participationKeys.lists() })` with `qc.invalidateQueries({ queryKey: ['registration', 'list'] })` (or import `regKeys` and use `regKeys.lists()`), because `useOrganizers` reads from the `registration` query namespace.

**FIX-02 · Excel download raw fetch does not retry on expired token**
- **Risk:** Silent download failure if access token expires between page load and download click; user sees nothing or a corrupt file.
- **Module:** reports
- **File:** `modules/reports/services/reports.service.ts:38–50`
- **Fix prompt:** Replace raw `fetch(path)` in `downloadExcelBlob` with a call through the typed `api.GET` client (returning `Blob` response type), or intercept the 401 response and call `tryRefresh()` + retry before throwing, mirroring the logic in `core/api/client.ts:52–68`.

**FIX-03 · `listSubmissions` no RBAC scoping param**
- **Risk:** If route guard for submissions page fails or is bypassed, all federation participation data is visible to any authenticated user.
